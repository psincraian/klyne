Basic Setup

Initialize Klyne in your package's __init__.py file.

Minimal Integration

__init__.py
import klyne
klyne.init(
api_key='klyne_abc123def456',
project='your-package-name'
)
That's it!
Your package will now automatically send analytics data when imported by users. API keys are public identifiers (like Google Analytics IDs) and are safe to include directly in your code.

Configuration Options

Available Parameters

Parameter Type Required Description
api_key string Yes Your package's public API key (safe to include in code)
project string Yes Your package name (must match PyPI name)
package_version string Optional Current version of your package
enabled boolean Optional Enable/disable tracking (default: true)

Best Practices

Do

  • • Initialize in your package's __init__.py
  • • Include API keys directly in your code
  • • Test in development before releasing
  • • Respect user privacy and opt-out settings

Don't

  • • Initialize multiple times
  • • Block on analytics calls
  • • Collect personal user data
  • • Override user opt-out preferences

Frequently Asked Questions

How does Klyne track package usage?

Klyne automatically sends anonymous usage data when your package is imported. This includes system information like OS type, Python version, and geographic region (country-level only).

What data does Klyne collect?

Klyne collects only essential analytics data: operating system, Python version, package version, and approximate geographic location. We never collect personal information, source code, or sensitive data.

Will Klyne slow down my package?

No. Klyne is designed to have minimal performance impact. Analytics data is sent asynchronously in the background and won't block your package's functionality.

Troubleshooting

Common Issues

Analytics not showing in dashboard

  • Verify your API key is correct
  • Check that the project name matches your PyPI package name
  • Ensure your package is being imported by users
  • Wait 5-10 minutes for data to appear (there may be a delay)

Import errors

  • Make sure Klyne is installed: pip install klyne
  • Check your Python version is 3.7 or higher
  • Verify your API key format starts with "klyne_"
  • Try wrapping the initialization in a try-except block

Performance concerns

  • Klyne runs asynchronously and shouldn't affect performance
  • You can disable tracking in development with enabled=False
  • Analytics are sent in background threads without blocking your code

API Reference

Analytics Endpoint

Submit Analytics Event

POST
POST /api/analytics

Submit analytics events for your package. This endpoint is used automatically by the Klyne SDK.

Headers
Header Value Description
Authorization Bearer YOUR_API_KEY Your package's API key
Content-Type application/json Request content type
Request Body
JSON
{
"event_type": "package_import",
"package_name": "your-package-name",
"package_version": "1.0.0",
"python_version": "3.9.7",
"os_name": "Linux",
"timestamp": "2024-01-15T10:30:00Z"
}
Response
200 OK - Event recorded successfully
SDK Integration
The Klyne SDK handles all API interactions automatically. You typically don't need to call this endpoint directly. API keys are public tracking identifiers, similar to Google Analytics IDs.
